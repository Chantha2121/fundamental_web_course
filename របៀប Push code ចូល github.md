## **របៀប Push គម្រោងទៅកាន់ GitHub**

មុននឹងចាប់ផ្ដើម សូមប្រាកដថាអ្នកបានបង្កើត **Repository** ថ្មីមួយនៅលើ GitHub រួចរាល់ហើយ។

### **ជំហានទី ១៖ ចាប់ផ្ដើម Git ក្នុងម៉ាស៊ីន (Initialize)**

ប្រសិនបើគម្រោងរបស់អ្នកមិនទាន់មាន Git ទេ សូមបើក Terminal ឬ Command Prompt ក្នុង Folder គម្រោង រួចវាយពាក្យបញ្ជា៖

```bash
# ចាប់ផ្ដើមបង្កើត Git ក្នុង Folder គម្រោង
git init

# បន្ថែម File ទាំងអស់ទៅក្នុង Staging Area
git add .

# រក្សាទុកការផ្លាស់ប្តូរ (Commit) ជាមួយសារបញ្ជាក់
git commit -m "បញ្ចូលគម្រោងដំបូង"

```

### **ជំហានទី ២៖ ភ្ជាប់ទៅកាន់ GitHub (Remote Add)**

អ្នកត្រូវប្រាប់ Git ឱ្យដឹងថាត្រូវ Push ទៅកាន់ Repository មួយណា ដោយប្រើ Link ដែលអ្នកបានចម្លងពី GitHub៖

```bash
# ភ្ជាប់ Repository មូលដ្ឋាន ទៅកាន់ GitHub
git remote add origin https://github.com/ឈ្មោះអ្នកប្រើ/ឈ្មោះគម្រោង.git

# ប្តូរឈ្មោះ Branch ទៅជា main (ជាស្ដង់ដារបច្ចុប្បន្ន)
git branch -M main

```

### **ជំហានទី ៣៖ បញ្ជូនទៅ GitHub (Push)**

នេះគឺជាជំហានចុងក្រោយដើម្បីផ្ទេរ File ពីម៉ាស៊ីនរបស់អ្នកទៅកាន់ GitHub៖

```bash
# បញ្ជូន Code ទៅកាន់ Branch main
git push -u origin main

```

### **តារាងពាក្យបញ្ជាដែលប្រើញឹកញាប់**

| ពាក្យបញ្ជា | អត្ថន័យ |
| --- | --- |
| `git status` | ពិនិត្យមើលស្ថានភាព File (តើមួយណាបាន Add ឬមិនទាន់) |
| `git log` | មើលប្រវត្តិនៃការ Commit កន្លងមក |
| `git pull` | ទាញយក Code ចុងក្រោយពី GitHub មកម៉ាស៊ីនវិញ |

> **ចំណាំ៖** ប្រសិនបើអ្នកជួបបញ្ហា **Merge Conflict** (មានការជាន់គ្នានៃកូដ) អ្នកត្រូវដោះស្រាយការជាន់គ្នានោះក្នុង File នីមួយៗ រួចទើបអាច Commit និង Push ម្ដងទៀតបាន។
